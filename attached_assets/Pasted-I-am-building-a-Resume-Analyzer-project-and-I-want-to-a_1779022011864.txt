== FEATURE OVERVIEW ==
When the user clicks the "Start Interview" button, show a modal 
with two options:
1. Text-Based Interview
2. Voice-Based Interview

== INTERVIEW SETTINGS MODAL ==
Before starting the interview, show a settings screen with the 
following options:

GENERAL SETTINGS (for both Text and Voice):
- Role: Text input or dropdown (if not already selected from resume, 
  ask here. If already selected from resume, pre-fill it)
- Company Type:
  * Startup
  * Corporate
  * FAANG
- Interviewer Tone:
  * Friendly
  * Neutral
  * Strict
- Duration:
  * Quick (10 min / ~5 questions)
  * Standard (20 min / ~10 questions)

VOICE SETTINGS (only shown if Voice-Based Interview is selected):
- Gender:
  * Male
  * Female
- Accent:
  * US
  * UK
  * Australian
- Speaking Speed:
  * Slow
  * Normal
  * Fast
All voice settings use Web Speech Synthesis API (free, browser 
built-in, no extra API needed). Apply the selected gender, accent, 
and speed to the SpeechSynthesisUtterance configuration.

== TEXT-BASED INTERVIEW ==
- Conversational chat UI (not a form)
- AI acts as a professional interviewer
- AI greets the user and introduces itself
- Starts with warm-up question (Tell me about yourself)
- Asks role-specific and resume-specific questions
- Questions should be tailored based on:
  * Selected Role
  * Selected Company Type (FAANG questions are harder and 
    more technical, Startup questions are more versatile, 
    Corporate questions are more structured)
- After each user answer, AI asks an intelligent follow-up 
  question based on what the user said
- Then moves to the next main question
- Uses STAR method awareness in follow-ups
- Ends with "Do you have any questions for us?"
- Then shows "Interview Complete" screen and generates report

== VOICE-BASED INTERVIEW ==
- Same flow as Text-Based Interview
- AI question is read aloud using Web Speech Synthesis API
  (free, browser built-in, no extra API needed)
- Apply voice settings selected by user:
  * Gender: Male or Female voice
  * Accent: US / UK / Australian
  * Speed: Slow (0.75) / Normal (1.0) / Fast (1.25)
- User speaks their answer using Web Speech Recognition API
  (free, browser built-in, no extra API needed)
- Microphone should be OFF while AI is speaking
- Microphone should turn ON automatically after AI finishes speaking
- User's spoken answer is converted to text and sent to Gemini
- Show live transcription of what user is saying on screen
- Show the full conversation history as text on screen as well
- Show a visual speaking indicator when AI is talking
- Show a visual mic indicator when mic is active and listening
